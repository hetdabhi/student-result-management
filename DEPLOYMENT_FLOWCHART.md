## ğŸš¨ Common Pitfalls to Avoid

1. **Forgetting to deploy security rules**
   - Always run: `firebase deploy --only firestore:rules`

2. **Not setting custom claims**
   - Users must have role claims set
   - Must logout/login after setting claims

3. **Missing Firestore documents**
   - Every user needs a document in users collection
   - Document ID must match Firebase Auth UID

4. **Wrong Firebase config**
   - Double-check all values in firebase-config.js
   - No extra spaces or quotes

5. **Service account key not in scripts folder**
   - Must be in scripts/serviceAccountKey.json
   - Never commit to Git!
